모두를 위한 딥러닝-딥러닝의 기본 7. 머신러닝의 실용과 팁

January 25, 2020

7-1. 학습 rate, Overfitting, 그리고 일반화

running rate \(\alpha\)

  • 적절한 값을 정해주는 것이 중요
  • 만약 값이 너무 크다면 값이 발산함 (overshooting)
  • 만약 값이 너무 작다면 학습이 느리게 이루어짐

x의 선처리(nomalization)

  • x 요소 사이에 값에 큰 차이가 있을 경우 학습이 제대로 이루어지지 않음
  • standardization : \(x_{j}`=\frac{x_{j}-\mu_{j}}{\sigma _{j}}\) (뮤 : 평균 / 시그마 : 분산)

overfitting

  • 학습 데이터에만 너무 딱맞는 모델이 만들어지는 현상
  • 학습 데이터에는 잘 맞지만 실제 데이터나 테스트 데이터에는 잘 맞지 않음

overfitting의 해결법

  • 학습 데이터를 더 많이 준비
  • 중복된 특징의 갯수를 줄임
  • 일반화 시킴 (너무 큰 가중치를 쓰지 않음)

일반화 방법

  • loss의 수정

\(L=\frac{1}{N}\sum_{i}D(S(wx_{i}+b),L_{i})+\lambda\sum W^{2}\)

로스에 가중치의 총 합을 포함시켜 가중치가 최소가 될 수 있도록 조절

7-2. Training/Testing 데이타 셋

모델의 평가 방법

  • 매우 나쁜 방법

    • 트레이닝셋과 비교

  • 좋은 방법

    • 트레이닝셋으로 학습 후 테스트셋과 비교

  • 알파와 람다값을 조정하기 위해 Training셋에서 Validation셋을 분리하여 사용

online learning

  • 트레이닝셋을 분리하여 학습하는 방법
  • 새롭게 데이터 셋이 들어오는 상황에서 유용함
