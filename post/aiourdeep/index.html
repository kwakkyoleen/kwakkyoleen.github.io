머신러닝

  • explict program 의 제한을 벗어난 프로그램. 개발자가 정해준 일정한 로직을 따르지 않고 학습한 내용을 토대로 움직이는 프로그램.

머신러닝의 종류

  • supervised learning : training set(레이블들이 정해져 있는 데이터)을 가지고 학습을 하는 방법 - 예시) 개와 고양이를 구분하는 프로그램, 개와 고양이로 레이블이 구분된 자료로 학습
  • unsupervied learning : un-labeled data(데이터가 레이블이 주어지지 않은 경우)를 보고 유사한것들을 구분하여 학습하는 방법 - 예시) 구글 뉴스의 그룹화
  • 이번 강의에서는 supervised learning을 중점으로 볼것임

ML에서의 일반적이 문제 유형

  • 레이블링된 이미지를 보고 학습
  • 스펨메일로 구분된, 또는 구분되지 않은 메일을 보고 학습
  • 시험 준비 시간과 시험 점수가 주어진 데이터를 보고 학습

training data set

  • 답이라고 할 수 있는 Y값과 특징이라고 할 수 있는 X를 가지고 있는 레이블화된 데이터.

TenserFlow에대한 간략한 설명

트레이닝 데이터를 간단하게 나타낸 그래프

  • X : 예측을 하기 위한 기본적인 자료 (특징)
  • Linear regression : 선형적인 모델이 데이터에 맞을거라 가설을 세우고 선을 만들어 예측하는 방법 - 예시) 공부량과 성적, 집 크기와 가격 등등
  • H(x) = Wx + b -> 가장 적절한 W와 b 값을 찾아냄 (Hypothesis)

Cost function (Loss function)

  • 실제 데이터와 예측하여 만든 선과의 거리를 비교하여 얼마나 가설이 적절한가 판단
  • \( H(x)-y \) -> 좋은 모델은 아님 (차이가 음수일 수 도 있음)
  • \( (H(x)-y)^{2} \) -> 차이가 양수로 나타남 -> cost function
  • 일반화 모델 : \(cost(W,b)=\frac{1}{m}\sum_{i=1}^{m}(H(x^{(i)})-y^{(i)})^{2}\) (m : 데이터 갯수)
  • 최종 목적 : \(minimize_{(W, b)} cost(W,b)\) -> cost를 작게 만드는 것

2-L. TensorFlow로 Linear Regression 구현

operation으로 그래프 만들기

\(H(x)=Wx+b\) 의 경우

x_train = [1, 2, 3]
y_train = [1, 2, 3]
varia = [W, b]

W = tf.Variable(tf.random.normal(shape=[1]),name = 'weight') # variable : 학습하며 변경 가능한 tf 타입
b = tf.Variable(tf.random.normal(shape=[1]), name = 'bias')

hypothesis = x_train * W + b

\(cost(W,b)=\frac{1}{m}\sum_{i=1}^{m}(H(x^{(i)})-y^{(i)})^{2}\) 의 경우

cost = tf.reduce_mean(tf.square(hypothesis - y_train)) # reduce_mean : 평균을 내주는 함수

GradientDescent

optimizer = tf.keras.optimizers.SGD(learning_rate=0.01)

def hyp(x_,w_,b_): # 현재 hypothesis(W, b) 를 구해주는 함수
    return x_*w_+b_

def loss(pre_y, des_y): #현재 cost를 구해주는 함수
    return tf.math.reduce_mean(tf.square(pre_y - des_y))

for i in range(10000):
    with tf.GradientTape() as t: #cost를 줄이며 경사를 하강시키는 부분
        current_loss = loss(hyp(x_train,W,b),y_train)   
    grads = t.gradient(current_loss,varia)
    optimizer.apply_gradients(zip(grads, varia))

3. cost의 최소화

cost를 줄이는 방법을 알고 linear regression을 완성시켜보자

hypothesis와 cost함수 다시보기

\(H(x)=Wx+b\)

\(cost(W,b)=\frac{1}{m}\sum_{i=1}^{m}(H(x^{(i)})-y^{(i)})^{2}\)

b = 0인 버전 : \(cost(W,b)=\frac{1}{m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})^{2}\)

cost 함수란 무엇일까?

트레이닝 데이터

X Y
1 1
2 2
3 3
  • W = 1, cost = \(\frac{1}{m}((1\times 1-1)^{2}+(1\times 2-2)^{2}+(1\times 3-3)^{2})=0\)
  • W = 0, cost = \(\frac{1}{m}((0\times 1-1)^{2}+(0\times 2-2)^{2}+(0\times 3-3)^{2})=4.67\)
  • W = 2, cost = 4.67

W값의 변화에 따른 cost그래프

  • 목표 : cost가 최소가 되는 W를 찾는 것

Gradient descent algorithm (경사 하강 알고리즘)

  • cost를 최소화 하는 함수
  • 많은 최소화 문제에 사용됨
  • cost(W, b)가 주어졌을 때 cost를 최소화 하는 W, b를 찾음
  • 더 일반적인 함수에도 적용 가능

경사 하강 알고리즘의 동작 방식

  • 경사가 있는 방향으로 내려감 -> 언젠가는 바닥에 도착
  • cost 함수에 미분을 적용
    • 변형된 cost 함수 : \(cost(W,b)=\frac{1}{2m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})^{2}\)
    • 정리 전 :\(W:=W-\alpha \frac{\partial }{\partial W}cost(W)\) - (\(\alpha \) = running rate)
    • 정리 후 : \(W:=W-\alpha \frac{1}{m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})x^{(i)}\)

Convex function

위 그림과 같이 시작점이 어디냐에 따라서 도착점이 달라 질 수가 있음

하지만 우리가 사용하는 cost함수는 위 그림과 같은 그릇 모양의 Convex function이라 어느 위치에서 시작하든지 항상 바닥에 도착하게 된다. 즉 cost함수를 설계할 때 그 함수가 Convex함수인지 꼭 확인해야 한다.

3-L. cost 최소화의 Tensorflow 구현

import tensorflow as tf
import matplotlib.pyplot as plt

x_train = [1, 2, 3]
y_train = [1, 2, 3]

W_val = []
cost_val = []

def hyp(x_,w_):
    return x_*w_

def cost(pre_y, des_y):
    return tf.math.reduce_mean(tf.square(pre_y - des_y))

for i in range(-30,50):
    W = tf.Variable(i*0.1)
    curr_hyp = hyp(x_train,W)
    curr_W = W.numpy()
    curr_cost = cost(curr_hyp,y_train).numpy()
    W_val.append(curr_W)
    cost_val.append(curr_cost)
    
print("W = ",W_val,", cost = ",cost_val)

plt.plot(W_val,cost_val)
plt.show()

W 값 변화에 따른 cost 변화를 보기 위해 위와 같은 코드를 작성했다.

위 그래프는 앞의 코드의 출력이며 가로축은 W, 세로축은 cost를 나타낸다.

그래프 분석

  • W가 1 이상일 때
    • 기울기는 양수
    • W는 작아져야 함
  • W가 1 이하일 때
    • 기울기는 음수
    • W는 커져야 함
  • 따라서 산술적 계산을 적용하면 W에 기울기를 정리한 값을 빼주면 올바른 결과가 나온다는 결론이 나온다.
    • \(W:=W-\alpha \frac{1}{m}\sum_{i=1}^{m}(Wx^{(i)}-y^{(i)})x^{(i)}\)

이를 Tensorflow 코드에 적용하면

learning_rate = 0.1
gradient = tf.math.reduce_mean((W*X-Y)*X)
W = tf.subtract(W, learning_rate*gradient)

그리고 이 내용을 앞선 코드에 적용하여 코드를 다시 짜보면

import tensorflow as tf

y_train = [1, 2, 3]

    return x_*w_

def cost(pre_y, des_y):
    return tf.math.reduce_sum(tf.square(pre_y - des_y))

for i in range(25):

    gradient = tf.math.reduce_mean((W*x_train-y_train)*x_train)
    W = tf.subtract(W, learning_rate*gradient)
    print(cost(hyp(x_train,W),y_train).numpy(),W.numpy())

varia = [W, b]

# ...

    return x_*w_+b_

x_train = [1, 2, 3]
y_train = [1, 2, 3]

W = tf.Variable(5.0 ,name = 'weight')
varia = [W]

optimizer = tf.keras.optimizers.SGD(learning_rate=0.02)

def hyp(x_,w_):
    return x_*w_

def cost(pre_y, des_y):
    return tf.math.reduce_sum(tf.square(pre_y - des_y))

for i in range(10):
    with tf.GradientTape() as t:
        current_cost = cost(hyp(x_train,W),y_train)   
    grads = t.gradient(current_cost,varia)
    optimizer.apply_gradients(zip(grads, varia))
    print(cost(hyp(x_train,W),y_train).numpy(),W.numpy())

  • 기본연산

    • tf.add: 덧셈
    • tf.subtract: 뺄셈
    • tf.multiply: 곱셈
    • tf.divide: 나눗셈
    • tf.pow: n-제곱
    • tf.negative: 음수 부호
    • tf.abs: 절대값
    • tf.sign: 부호
    • tf.round: 반올림
    • tf.ceil: 올림
    • tf.floor: 내림
    • tf.square: 제곱
    • tf.sqrt: 제곱근
    • tf.maximum: 두 텐서의 각 원소에서 최댓값만 반환.
    • tf.minimum: 두 텐서의 각 원소에서 최솟값만 반환.
    • tf.cumsum: 누적합
    • tf.cumprod: 누적곱

  • 차원축소 연산

    • tf.reduce_all: 설정한 축으로 이동하면서 and논리 연산을 수행한다.
    • tf.reduce_any: 설정한 축으로 이동하면서 or논리 연산을 수행한다.
    • tf.reduce_mean: 설정한 축의 평균을 구한다.
    • tf.reduce_max: 설정한 축의 최댓값을 구한다.
    • tf.reduce_min: 설정한 축의 최솟값을 구한다.
    • tf.reduce_prod: 설정한 축의 요소를 모두 곱한 값을 구한다.
    • tf.reduce_sum: 설정한 축의 요소를 모두 더한 값을 구한다.
    • tf.reduce_logsumexp: 설정한 축의 벡터가 N차원의 벡터 x라고 할 때 \(\mathrm{log}(\sum_{i=0}^{N}\mathrm{exp}x_{i})\)

  • 행렬 연산

    • tf.matmul: 내적
    • tf.linalg.inv: 역행렬
